Partie 3 : Exercices Pratiques (8 points)

Exercice 1 : Routes 2 pts

Écrivez les routes nécessaires pour un CRUD complet de ressource "Task" avec les spécifications suivantes :

  • GET /tasks → Liste toutes les tâches
  • GET /tasks/create → Formulaire de création
  • POST /tasks → Enregistre une nouvelle tâche
  • GET /tasks/{id} → Affiche une tâche
  • GET /tasks/{id}/edit → Formulaire d'édition
  • PUT /tasks/{id} → Met à jour une tâche
  • DELETE /tasks/{id} → Supprime une tâche

Utilisez la méthode la plus concise possible.

Exercice 2 : Migration 2 pts

Écrivez la méthode up() d'une migration pour créer la table "tasks" avec les colonnes suivantes :

  • id (clé primaire auto-incrémentée)
  • title (chaîne de 255 caractères, non null)
  • description (texte, nullable)
  • status (enum : 'pending', 'in_progress', 'completed', défaut 'pending')
  • priority (entier, défaut 1)
  • due_date (date, nullable)
  • user_id (clé étrangère vers users)
  • timestamps (created_at et updated_at)
Exercice 3 : Contrôleur 2 pts

Complétez la méthode store() du contrôleur TaskController pour :

  • Valider les données (title requis min 3 caractères, description optionnelle, due_date doit être une date future)
  • Créer la tâche en associant l'utilisateur connecté
  • Rediriger vers la liste des tâches avec un message de succès
Exercice 4 : Vue Blade 2 pts

Créez une vue Blade qui affiche la liste des tâches avec les éléments suivants :

  • Étend un layout "app"
  • Affiche un titre "Mes Tâches"
  • Boucle sur les tâches pour afficher le titre et le statut
  • Affiche "Aucune tâche" si la liste est vide
  • Inclut un lien pour créer une nouvelle tâche
