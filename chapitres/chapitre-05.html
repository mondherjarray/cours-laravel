Qu'est-ce qu'un ORM ?

Object-Relational Mapping

Définition

Un ORM (Object-Relational Mapping) est une technique qui permet de travailler avec une base de données en utilisant des objets PHP au lieu d'écrire du SQL directement.

Sans ORM (SQL brut)

$pdo->query("SELECT * FROM users WHERE status = 'active'");

Avec Eloquent ORM

User::where('status', 'active') ->get();
Avantages de l'ORM
  • Code plus lisible et maintenable
  • Protection contre les injections SQL
  • Indépendance vis-à-vis du SGBD
  • Relations entre tables simplifiées

Eloquent - L'ORM de Laravel

Un ORM élégant et puissant

Principes de base

Eloquent suit le pattern Active Record : chaque modèle correspond à une table et chaque instance à une ligne de cette table.

Mapping Eloquent

Concept PHP Concept Base de données
Classe Model (User) Table (users)
Instance de Model ($user) Ligne dans la table
Propriété ($user->name) Colonne de la table
Méthode ($user->save()) Requête SQL (INSERT/UPDATE)
Convention de nommage

Par défaut, le modèle User correspond à la table users (nom en minuscules au pluriel).

Configuration du Modèle

Propriétés importantes

class Project extends Model { // Nom de la table (si différent de la convention) protected $table = 'mes_projets'; // Clé primaire (si différente de 'id') protected $primaryKey = 'project_id'; // Désactiver les timestamps (created_at, updated_at) public $timestamps = false; // Champs autorisés pour l'assignation de masse protected $fillable = [ 'name', 'description', 'status' ]; // Champs protégés contre l'assignation de masse protected $guarded = ['id']; // Attributs cachés dans les tableaux/JSON protected $hidden = ['password']; }

Eager Loading

Optimiser les requêtes avec with()

Le problème N+1

Sans Eager Loading (N+1)

// 1 requête pour les projets // + N requêtes pour les tâches $projects = Project::all(); foreach ($projects as $p) { echo $p->tasks->count(); }

100 projets = 101 requêtes

Avec Eager Loading

// 2 requêtes seulement $projects = Project::with('tasks') ->get(); foreach ($projects as $p) { echo $p->tasks->count(); }

100 projets = 2 requêtes

Options avancées

// Multiple relations $projects = Project::with(['tasks', 'members'])->get(); // Relations imbriquées $projects = Project::with('tasks.comments')->get(); // Eager loading conditionnel $projects = Project::with(['tasks' => function($query) { $query->where('status', 'pending'); }])->get();

Soft Deletes

Suppression douce (corbeille)

Configuration

use Illuminate\Database\Eloquent\SoftDeletes; class Project extends Model { use SoftDeletes; // Active le soft delete } // Dans la migration $table->softDeletes(); // Ajoute la colonne deleted_at

Utilisation

// Suppression douce $project->delete(); // Marque deleted_at, ne supprime pas // Les requêtes normales excluent les supprimés Project::all(); // N'inclut pas les supprimés // Inclure les supprimés Project::withTrashed()->get(); // Seulement les supprimés Project::onlyTrashed()->get(); // Restaurer $project->restore(); // Suppression définitive $project->forceDelete();
