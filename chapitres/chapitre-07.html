Authentification vs Autorisation

Deux concepts fondamentaux de la sécurité

Authentification (AuthN)

Qui êtes-vous ?

  • Vérifie l'identité de l'utilisateur
  • Login / Mot de passe
  • Token, OAuth, 2FA
  • Répond à : "Est-ce bien vous ?"
Autorisation (AuthZ)

Que pouvez-vous faire ?

  • Vérifie les permissions
  • Rôles et droits d'accès
  • Gates et Policies
  • Répond à : "Avez-vous le droit ?"
Exemple concret

Authentification : L'utilisateur se connecte avec email/mot de passe

Autorisation : Seul l'admin peut supprimer des projets

Installation de Laravel Breeze

Étape par étape

Étapes d'installation

  1. Installer le package Breeze via Composer
    composer require laravel/breeze --dev
  2. Exécuter l'installation de Breeze
    php artisan breeze:install blade

    Options disponibles : blade, vue, react, api

  3. Exécuter les migrations
    php artisan migrate
  4. Installer les dépendances front-end et compiler
    npm install && npm run dev

Protéger les Routes avec Middleware

Restreindre l'accès aux utilisateurs connectés

Middleware 'auth'

Le middleware auth vérifie que l'utilisateur est connecté avant d'accéder à la route.

// routes/web.php use App\Http\Controllers\ProjectController; // Protéger une route unique Route::get('/projets', [ProjectController::class, 'index']) ->middleware('auth'); // Protéger un groupe de routes Route::middleware('auth')->group(function () { Route::get('/dashboard', [DashboardController::class, 'index']); Route::resource('projets', ProjectController::class); Route::resource('taches', TaskController::class); }); // Middleware 'guest' : uniquement pour les visiteurs non connectés Route::get('/login', [AuthController::class, 'showLogin']) ->middleware('guest');
Redirection automatique

Si un utilisateur non connecté accède à une route protégée, il est automatiquement redirigé vers /login.

Les Gates (Portes)

Autorisation basée sur des closures

Les Gates sont des fonctions simples qui déterminent si un utilisateur peut effectuer une action donnée.

Définir une Gate

// app/Providers/AppServiceProvider.php use Illuminate\Support\Facades\Gate; use App\Models\Project; use App\Models\User; public function boot(): void { // Gate pour vérifier si l'utilisateur peut modifier un projet Gate::define('update-project', function (User $user, Project $project) { return $user->id === $project->user_id; }); // Gate pour vérifier si l'utilisateur est admin Gate::define('admin', function (User $user) { return $user->role === 'admin'; }); }
Quand utiliser les Gates ?

Les Gates sont idéales pour des vérifications simples et globales qui ne sont pas liées à un modèle spécifique.

Utiliser les Gates

Vérifier les permissions

Dans un Contrôleur

use Illuminate\Support\Facades\Gate; public function update(Project $project) { // Méthode 1 : allows() if (Gate::allows('update-project', $project)) { // Autorisé } // Méthode 2 : denies() if (Gate::denies('update-project', $project)) { abort(403); } // Méthode 3 : authorize() - Exception auto Gate::authorize('update-project', $project); }

Dans une Vue Blade

{{-- Directive @can --}} @can('update-project', $project) <a href="{{ route('projets.edit', $project) }}"> Modifier </a> @endcan {{-- Directive @cannot --}} @cannot('update-project', $project) <p>Vous ne pouvez pas modifier ce projet</p> @endcannot {{-- Alternative avec @canany --}} @canany(['update-project', 'delete-project'], $project) <div class="actions">...</div> @endcanany

Les Policies (Politiques)

Autorisation organisée par modèle

Les Policies regroupent les règles d'autorisation liées à un modèle spécifique dans une classe dédiée.

Créer une Policy

php artisan make:policy ProjectPolicy --model=Project
// app/Policies/ProjectPolicy.php namespace App\Policies; use App\Models\Project; use App\Models\User; class ProjectPolicy { // Peut voir la liste des projets public function viewAny(User $user): bool { return true; // Tous les utilisateurs connectés } // Peut voir un projet spécifique public function view(User $user, Project $project): bool { return $user->id === $project->user_id; } // Peut créer un projet public function create(User $user): bool { return true; } }

Utiliser les Policies dans le Contrôleur

Méthode authorize()

// app/Http/Controllers/ProjectController.php class ProjectController extends Controller { public function index() { $this->authorize('viewAny', Project::class); return view('projets.index', ['projets' => Project::all()]); } public function show(Project $project) { $this->authorize('view', $project); return view('projets.show', compact('project')); } public function update(Request $request, Project $project) { $this->authorize('update', $project); $project->update($request->validated()); return redirect()->route('projets.show', $project); } public function destroy(Project $project) { $this->authorize('delete', $project); $project->delete(); return redirect()->route('projets.index'); } }

Gestion des Rôles Utilisateurs

Implémenter un système de rôles simple

1. Ajouter la colonne 'role' au modèle User

php artisan make:migration add_role_to_users_table
// Migration public function up(): void { Schema::table('users', function (Blueprint $table) { $table->enum('role', ['user', 'manager', 'admin']) ->default('user'); }); }

2. Ajouter des méthodes au modèle User

// app/Models/User.php class User extends Authenticatable { public function isAdmin(): bool { return $this->role === 'admin'; } public function isManager(): bool { return in_array($this->role, ['manager', 'admin']); } }

Middleware de Rôle Personnalisé

Protéger les routes par rôle

Créer le middleware

php artisan make:middleware RoleMiddleware
// app/Http/Middleware/RoleMiddleware.php namespace App\Http\Middleware; use Closure; use Illuminate\Http\Request; class RoleMiddleware { public function handle(Request $request, Closure $next, string ...$roles) { if (!$request->user() || !in_array($request->user()->role, $roles)) { abort(403, 'Accès non autorisé.'); } return $next($request); } }

Enregistrer et utiliser

// bootstrap/app.php (Laravel 11) ->withMiddleware(function (Middleware $middleware) { $middleware->alias(['role' => \App\Http\Middleware\RoleMiddleware::class]); }) // routes/web.php Route::get('/admin', [AdminController::class, 'index']) ->middleware(['auth', 'role:admin']);
