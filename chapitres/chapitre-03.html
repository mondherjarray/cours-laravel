1. Introduction aux Vues

La couche présentation de l'application

Définition
Une vue est un fichier qui contient le code HTML de votre application. Elle est responsable de l'affichage des données à l'utilisateur.

Emplacement des vues

resources/
    views/
        welcome.blade.php       # Page d'accueil par défaut
        layouts/
            app.blade.php       # Layout principal
        projets/
            index.blade.php     # Liste des projets
            show.blade.php      # Détail d'un projet
            create.blade.php    # Formulaire création
        components/
            alert.blade.php     # Composant réutilisable
Convention de nommage
  • Extension : .blade.php
  • Nom en minuscules avec tirets : mon-fichier.blade.php
  • Organiser par dossier selon le modèle : projets/, taches/

Créer et Afficher une Vue

Du contrôleur à l'affichage

Étape 1 : Créer le fichier de vue

Créez resources/views/projets/index.blade.php

<!DOCTYPE html> <html> <head> <title>Liste des Projets</title> </head> <body> <h1>Mes Projets</h1> <p>Bienvenue sur TaskFlow !</p> </body> </html>

Étape 2 : Retourner la vue depuis le contrôleur

// Dans ProjetController.php public function index() { // 'projets.index' = resources/views/projets/index.blade.php return view('projets.index'); }
Notation pointée
Le point . représente un séparateur de dossier :
  • view('projets.index') → views/projets/index.blade.php
  • view('admin.users.edit') → views/admin/users/edit.blade.php

Passer des Données à la Vue

Du contrôleur vers l'affichage

Méthode 1 : Tableau associatif

public function index() { $projets = ['Projet A', 'Projet B', 'Projet C']; $titre = 'Liste des projets'; return view('projets.index', [ 'projets' => $projets, 'titre' => $titre ]); }

return view('projets.index') ->with('projets', $projets) ->with('titre', $titre);

2. Le Moteur Blade

Afficher des données

Qu'est-ce que Blade ?
Blade est le moteur de templates de Laravel. Il permet d'écrire du PHP de manière plus simple et sécurisée dans les vues.

Afficher une variable

Syntaxe Blade

{{-- Échappé (sécurisé) --}} <h1>{{ $titre }}</h1> <p>{{ $description }}</p> {{-- Non échappé (HTML brut) --}} {!! $contenuHtml !!}

Équivalent PHP

// Échappé <h1><?= htmlspecialchars($titre) ?></h1> // Non échappé <?= $contenuHtml ?>

Valeur par défaut

{{-- Si $nom n'existe pas, affiche 'Invité' --}} <p>Bonjour, {{ $nom ?? 'Invité' }}</p>
Sécurité
Utilisez toujours {{ }} (double accolades) pour éviter les attaques XSS. N'utilisez {!! !!} que si vous êtes sûr que le contenu est sûr.

Directives Conditionnelles

@if, @else, @elseif

Structure if / else

@if($statut === 'termine') <span class="badge-success">Terminé</span> @elseif($statut === 'en_cours') <span class="badge-warning">En cours</span> @else <span class="badge-secondary">En attente</span> @endif

Directives de Boucle

@foreach, @for, @while

@foreach - La plus utilisée

<ul> @foreach($projets as $projet) <li>{{ $projet->nom }}</li> @endforeach </ul>

@forelse - Avec gestion du cas vide

@forelse($projets as $projet) <div class="card"> <h3>{{ $projet->nom }}</h3> <p>{{ $projet->description }}</p> </div> @empty <p>Aucun projet à afficher</p> @endforelse

Variable $loop

Dans une boucle, vous avez accès à la variable $loop :

@foreach($projets as $projet) <p> {{ $loop->iteration }}. {{ $projet->nom }} @if($loop->first) <span>(Premier)</span> @endif @if($loop->last) <span>(Dernier)</span> @endif </p> @endforeach

Propriétés de $loop

Informations sur l'itération en cours

Propriété Description Exemple
$loop->index Index actuel (commence à 0) 0, 1, 2...
$loop->iteration Numéro d'itération (commence à 1) 1, 2, 3...
$loop->remaining Itérations restantes 4, 3, 2, 1, 0
$loop->count Nombre total d'éléments 5
$loop->first Est-ce le premier élément ? true/false
$loop->last Est-ce le dernier élément ? true/false
$loop->even Itération paire ? true/false
$loop->odd Itération impaire ? true/false
$loop->parent Accès à la boucle parente Boucles imbriquées

Exemple : Alterner les couleurs

@foreach($projets as $projet) <tr class="{{ $loop->even ? 'bg-gray' : 'bg-white' }}"> <td>{{ $projet->nom }}</td> </tr> @endforeach

3. Layouts et Héritage

Éviter la répétition de code

Le problème

Sans layout, vous devez répéter le HTML de base dans chaque page :

page1.blade.php

<!DOCTYPE html> <html> <head> <title>Page 1</title> <link href="style.css"...> </head> <body> <nav>...</nav> <!-- Contenu page 1 --> <footer>...</footer> </body> </html>

page2.blade.php

<!DOCTYPE html> <html> <head> <title>Page 2</title> <link href="style.css"...> </head> <body> <nav>...</nav> <!-- Contenu page 2 --> <footer>...</footer> </body> </html>
Problème
  • Duplication de code HTML (head, nav, footer...)
  • Difficile à maintenir
  • Si on change le menu, il faut modifier TOUTES les pages
Solution : Les Layouts
Créer un fichier "template" avec la structure commune, et y injecter le contenu spécifique.

Utiliser le Layout

@extends et @section

Page qui hérite du layout

Fichier : resources/views/projets/index.blade.php

{{-- Hériter du layout --}} @extends('layouts.app') {{-- Définir le titre --}} @section('title', 'Liste des Projets') {{-- Définir le contenu principal --}} @section('content') <h1>Mes Projets</h1> @forelse($projets as $projet) <div class="card"> <h2>{{ $projet->nom }}</h2> <p>{{ $projet->description }}</p> <a href="/projets/{{ $projet->id }}">Voir détails</a> </div> @empty <p>Aucun projet trouvé.</p> @endforelse @endsection
Comment ça marche
  • @extends : Indique quel layout utiliser
  • @section : Définit le contenu à injecter dans un @yield
  • @endsection : Ferme la section

4. Composants Blade

Éléments réutilisables

Les composants permettent de créer des éléments d'interface réutilisables.

Créer un composant Alert

php artisan make:component Alert

Fichier créé : resources/views/components/alert.blade.php

{{-- Composant Alert --}} <div class="alert alert-{{ $type ?? 'info' }}"> @if(isset($title)) <strong>{{ $title }}</strong> @endif {{ $slot }} </div>

Utiliser le composant

{{-- Utilisation simple --}} <x-alert type="success"> Projet créé avec succès ! </x-alert> {{-- Avec titre --}} <x-alert type="danger" title="Erreur"> Une erreur est survenue. </x-alert>
Astuce
$slot contient le contenu entre les balises du composant.

5. Inclusion de Sous-vues

@include et @each

@include - Inclure une vue partielle

{{-- Inclure le header --}} @include('partials.header') {{-- Inclure avec des données --}} @include('partials.sidebar', ['menu' => $menuItems]) {{-- Inclure si existe --}} @includeIf('partials.optional') {{-- Inclure si condition vraie --}} @includeWhen($user->isAdmin, 'partials.admin-menu')

@each - Inclure pour chaque élément

{{-- Pour chaque projet, inclure la vue partials.projet-item --}} @each('partials.projet-item', $projets, 'projet') {{-- Avec vue pour le cas vide --}} @each('partials.projet-item', $projets, 'projet', 'partials.no-projets')

Fichier partiel exemple

resources/views/partials/projet-item.blade.php

<li> <strong>{{ $projet->nom }}</strong> - {{ $projet->statut }} </li>

Liens et Assets

URLs, routes, images, CSS

Générer des URLs

{{-- URL de base --}} <a href="{{ url('/projets') }}">Projets</a> {{-- Route nommée --}} <a href="{{ route('projets.index') }}">Liste</a> {{-- Route avec paramètre --}} <a href="{{ route('projets.show', $projet->id) }}">Voir</a> {{-- Route avec modèle (Route Model Binding) --}} <a href="{{ route('projets.show', $projet) }}">Voir</a>

Assets (CSS, JS, Images)

{{-- Fichiers dans public/ --}} <link href="{{ asset('css/app.css') }}" rel="stylesheet"> <script src="{{ asset('js/app.js') }}"></script> <img src="{{ asset('images/logo.png') }}" alt="Logo"> {{-- Avec Vite (Laravel 11) --}} @vite(['resources/css/app.css', 'resources/js/app.js'])
Conseil
Utilisez toujours route() au lieu d'écrire les URLs manuellement. Si vous changez l'URL de la route, les liens se mettent à jour automatiquement.

Formulaires et CSRF

Protection contre les attaques

CSRF - Cross-Site Request Forgery
Laravel protège automatiquement vos formulaires contre les attaques CSRF. Vous devez ajouter @csrf dans chaque formulaire.

Formulaire simple

<form action="{{ route('projets.store') }}" method="POST"> @csrf <label>Nom du projet</label> <input type="text" name="nom" required> <label>Description</label> <textarea name="description"></textarea> <button type="submit">Créer</button> </form>

Méthode PUT/DELETE (spoofing)

{{-- HTML ne supporte que GET et POST --}} {{-- @method simule PUT, PATCH, DELETE --}} <form action="{{ route('projets.update', $projet) }}" method="POST"> @csrf @method('PUT') {{-- Champs du formulaire --}} </form> <form action="{{ route('projets.destroy', $projet) }}" method="POST"> @csrf @method('DELETE') <button>Supprimer</button> </form>
