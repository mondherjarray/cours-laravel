1. Qu'est-ce qu'une Route ?

Le point d'entrée de votre application

Définition
Une route est une règle qui dit à Laravel : "Quand un utilisateur visite cette URL, exécute ce code".

Comment ça fonctionne ?

Utilisateur

Tape une URL

→
Routeur

Trouve la route

→
Contrôleur

→
Réponse

Page HTML

Exemple simple

Quand l'utilisateur visite http://monsite.com/bonjour

// routes/web.php Route::get('/bonjour', function () { return 'Bonjour le monde !'; });

Les Fichiers de Routes

Où définir vos routes ?

Les routes sont définies dans le dossier routes/

Fichier Utilisation Accès
web.php Pages web classiques (HTML) Sessions, cookies, CSRF
api.php API REST (JSON) Authentification par token
console.php Commandes Artisan personnalisées Terminal uniquement

Routes Web (web.php)

// Accès : http://site.com/ Route::get('/', function () { return view('welcome'); });

Routes API (api.php)

// Accès : http://site.com/api/users Route::get('/users', function () { return User::all(); });
Note Laravel 11
Dans Laravel 11, les routes API sont préfixées automatiquement par /api.

2. Les Méthodes HTTP

GET, POST, PUT, DELETE...

Chaque route répond à une méthode HTTP spécifique :

Méthode Utilisation Exemple
GET Lire / Afficher des données Afficher la liste des projets
POST Créer une nouvelle ressource Créer un nouveau projet
PUT/PATCH Modifier une ressource existante Modifier un projet
DELETE Supprimer une ressource Supprimer un projet

Syntaxe Laravel

// Route GET - Afficher Route::get('/projets', [ProjetController::class, 'index']); // Route POST - Créer Route::post('/projets', [ProjetController::class, 'store']); // Route PUT - Modifier Route::put('/projets/{id}', [ProjetController::class, 'update']); // Route DELETE - Supprimer Route::delete('/projets/{id}', [ProjetController::class, 'destroy']);

Paramètres de Route

Routes dynamiques avec {paramètre}

Les paramètres permettent de capturer des valeurs dans l'URL.

Paramètre obligatoire

// URL : /projets/5 Route::get('/projets/{id}', function ($id) { return "Afficher le projet numéro : " . $id; }); // URL : /utilisateurs/ahmed Route::get('/utilisateurs/{nom}', function ($nom) { return "Bonjour " . $nom; });

Paramètre optionnel

Ajoutez ? et une valeur par défaut :

// URL : /salut ou /salut/Ahmed Route::get('/salut/{nom?}', function ($nom = 'Invité') { return "Bonjour " . $nom; });

Plusieurs paramètres

// URL : /projets/5/taches/12 Route::get('/projets/{projet}/taches/{tache}', function ($projet, $tache) { return "Projet $projet, Tâche $tache"; });

Contraintes sur les Paramètres

Valider le format des paramètres

Vous pouvez limiter le format des paramètres avec des expressions régulières :

Méthode where()

// L'id doit être un nombre Route::get('/projets/{id}', function ($id) { // ... })->where('id', '[0-9]+'); // Le nom doit contenir uniquement des lettres Route::get('/utilisateurs/{nom}', function ($nom) { // ... })->where('nom', '[A-Za-z]+');

Méthodes raccourcies

// whereNumber : uniquement des chiffres Route::get('/projets/{id}', ...)->whereNumber('id'); // whereAlpha : uniquement des lettres Route::get('/users/{nom}', ...)->whereAlpha('nom'); // whereAlphaNumeric : lettres et chiffres Route::get('/posts/{slug}', ...)->whereAlphaNumeric('slug'); // whereUuid : format UUID Route::get('/orders/{uuid}', ...)->whereUuid('uuid');
Conseil
Utilisez toujours des contraintes pour sécuriser vos routes et éviter les erreurs.

3. Routes Nommées

Donner un nom à vos routes

Nommer une route permet de l'utiliser facilement dans le code sans écrire l'URL complète.

Définir une route nommée

Route::get('/projets', [ProjetController::class, 'index']) ->name('projets.index'); Route::get('/projets/{id}', [ProjetController::class, 'show']) ->name('projets.show'); Route::post('/projets', [ProjetController::class, 'store']) ->name('projets.store');

Utiliser une route nommée

Dans un contrôleur

// Redirection return redirect() ->route('projets.index'); // Avec paramètre return redirect() ->route('projets.show', ['id' => 5]);

Dans une vue Blade

{{-- Lien simple --}} <a href="{{ route('projets.index') }}"> Voir les projets </a> {{-- Avec paramètre --}} <a href="{{ route('projets.show', 5) }}"> Projet 5 </a>
Avantage
Si vous changez l'URL, vous n'avez qu'à modifier la route. Tous les liens utilisant route('nom') seront mis à jour automatiquement.

Groupes de Routes

Organiser les routes avec des préfixes

Les groupes permettent d'appliquer des options communes à plusieurs routes.

Préfixe d'URL

Route::prefix('admin')->group(function () { // URL : /admin/dashboard Route::get('/dashboard', [AdminController::class, 'dashboard']); // URL : /admin/utilisateurs Route::get('/utilisateurs', [AdminController::class, 'users']); // URL : /admin/parametres Route::get('/parametres', [AdminController::class, 'settings']); });

Préfixe de nom

Route::name('admin.')->prefix('admin')->group(function () { // Nom : admin.dashboard Route::get('/dashboard', ...)->name('dashboard'); // Nom : admin.users Route::get('/utilisateurs', ...)->name('users'); });
Application TaskFlow
Nous utiliserons des groupes pour séparer les routes publiques et les routes admin.

4. Les Contrôleurs

Organiser la logique de l'application

Définition
Un contrôleur est une classe PHP qui regroupe la logique de traitement pour un ensemble de routes liées.

Pourquoi utiliser des contrôleurs ?

❌ Sans contrôleur

// routes/web.php Route::get('/projets', function () { $projets = Projet::all(); return view('projets.index', ['projets' => $projets]); }); // Code dans le fichier routes ! // Difficile à maintenir...

✅ Avec contrôleur

// routes/web.php Route::get('/projets', [ProjetController::class, 'index']); // Logique dans le contrôleur // Code propre et organisé !

Emplacement des contrôleurs

app/
    Http/
        Controllers/
            Controller.php          # Classe de base
            ProjetController.php    # Vos contrôleurs
            TacheController.php

Créer un Contrôleur

Avec la commande Artisan

Commande de création

php artisan make:controller ProjetController

Résultat : app/Http/Controllers/ProjetController.php

<?php namespace App\Http\Controllers; class ProjetController extends Controller { // Afficher la liste des projets public function index() { $projets = Projet::all(); return view('projets.index', ['projets' => $projets]); } // Afficher un seul projet public function show($id) { $projet = Projet::findOrFail($id); return view('projets.show', ['projet' => $projet]); } // Afficher le formulaire de création public function create() { return view('projets.create'); } }

Lier Route et Contrôleur

La syntaxe moderne de Laravel

Importer le contrôleur

// routes/web.php use App\Http\Controllers\ProjetController;

Syntaxe recommandée (Laravel 8+)

// [Classe::class, 'méthode'] Route::get('/projets', [ProjetController::class, 'index']); Route::get('/projets/create', [ProjetController::class, 'create']); Route::get('/projets/{id}', [ProjetController::class, 'show']); Route::post('/projets', [ProjetController::class, 'store']);

Ancienne syntaxe (déconseillée)

// Chaîne de caractères - à éviter Route::get('/projets', 'ProjetController@index');
Important

Utilisez toujours la syntaxe avec ::class. Elle permet à votre IDE de détecter les erreurs et facilite la refactorisation.

Contrôleur de Ressource

CRUD complet en une seule commande

Un contrôleur de ressource génère automatiquement toutes les méthodes CRUD.

Créer un contrôleur ressource

php artisan make:controller ProjetController --resource

Les 7 méthodes générées

Méthode URL Action
index() GET /projets Afficher la liste
create() GET /projets/create Formulaire de création
store() POST /projets Enregistrer un nouveau
show($id) GET /projets/{id} Afficher un seul
edit($id) GET /projets/{id}/edit Formulaire de modification
update($id) PUT /projets/{id} Mettre à jour
destroy($id) DELETE /projets/{id} Supprimer

Route Resource

7 routes en une seule ligne

Déclarer une route ressource

// Une seule ligne crée 7 routes ! Route::resource('projets', ProjetController::class);

Vérifier les routes créées

php artisan route:list

Limiter les routes

// Seulement ces méthodes Route::resource('projets', ProjetController::class) ->only(['index', 'show']); // Toutes sauf celles-ci Route::resource('projets', ProjetController::class) ->except(['destroy']);

Route API Resource (sans create/edit)

// Pour les API - 5 routes sans formulaires Route::apiResource('projets', ProjetController::class);

Injection de Dépendances

Récupérer automatiquement les objets

Laravel peut automatiquement injecter des objets dans vos méthodes de contrôleur.

Injecter la Request

use Illuminate\Http\Request; public function store(Request $request) { // Accéder aux données du formulaire $nom = $request->input('nom'); $email = $request->input('email'); // Ou récupérer toutes les données $data = $request->all(); }

Route Model Binding

Laravel peut récupérer automatiquement un modèle depuis la base de données :

// Route Route::get('/projets/{projet}', [ProjetController::class, 'show']); // Contrôleur - $projet est automatiquement récupéré ! public function show(Projet $projet) { // $projet contient déjà le projet avec l'id de l'URL return view('projets.show', ['projet' => $projet]); }
Astuce
Le nom du paramètre dans la route doit correspondre au nom de la variable dans la méthode.

5. Les Middlewares

Filtrer les requêtes HTTP

Définition
Un middleware est un filtre qui examine les requêtes HTTP avant qu'elles n'atteignent le contrôleur.
Requête
→
Middleware

Vérification

→
Contrôleur

Exemples de middlewares

  • auth : Vérifie que l'utilisateur est connecté
  • guest : Vérifie que l'utilisateur n'est PAS connecté
  • verified : Vérifie que l'email est vérifié
  • throttle : Limite le nombre de requêtes

Utiliser un middleware

// Sur une route Route::get('/dashboard', [DashboardController::class, 'index']) ->middleware('auth'); // Sur un groupe Route::middleware('auth')->group(function () { Route::get('/projets', ...); Route::get('/taches', ...); });
