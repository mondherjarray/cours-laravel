1. Pourquoi utiliser un Framework ?

Les limites du développement PHP classique

Le problème avec PHP natif

Quand on développe un site web en PHP sans framework, on rencontre plusieurs difficultés :

❌ Sans Framework

  • Code mélangé (PHP + HTML)
  • Pas de structure claire
  • Sécurité difficile à gérer
  • Répétition du même code
  • Maintenance compliquée
  • Travail en équipe difficile

✅ Avec Framework

  • Code bien organisé
  • Structure MVC claire
  • Sécurité intégrée
  • Code réutilisable
  • Maintenance facile
  • Collaboration simplifiée
📝 Définition
Un framework est un ensemble d'outils et de règles qui facilitent le développement d'applications. Il fournit une structure de base et des fonctionnalités prêtes à l'emploi.

Le framework PHP le plus populaire au monde

3. L'Architecture MVC

Modèle - Vue - Contrôleur

MVC est un patron de conception (design pattern) qui sépare l'application en 3 parties distinctes :

📦 Modèle

Données
Base de données

↔
⚙️ Contrôleur

Logique
Traitement

↔
🖥️ Vue

Affichage
Interface

💡 Pourquoi séparer le code ?
  • Maintenance : Modifier une partie sans toucher aux autres
  • Réutilisation : Utiliser le même modèle avec différentes vues
  • Travail en équipe : Un développeur par couche
  • Tests : Tester chaque partie séparément

Les 3 Composants du MVC

Rôle de chaque couche

📦 Le Modèle (Model)

  • Représente les données de l'application
  • Gère la communication avec la base de données
  • Contient les règles métier (validation, calculs...)
  • Dans Laravel : fichiers dans app/Models/

🖥️ La Vue (View)

  • Gère l'affichage à l'utilisateur
  • Contient le code HTML et la mise en page
  • N'effectue aucun traitement logique
  • Dans Laravel : fichiers .blade.php dans resources/views/

⚙️ Le Contrôleur (Controller)

  • Fait le lien entre le Modèle et la Vue
  • Reçoit les requêtes de l'utilisateur
  • Traite les données et choisit quelle vue afficher
  • Dans Laravel : fichiers dans app/Http/Controllers/

Cycle d'une Requête Laravel

De la demande à la réponse

Quand un utilisateur visite une page, voici ce qui se passe :

  1. L'utilisateur tape une URL dans son navigateur
    Exemple : http://monsite.com/utilisateurs
  2. Le Routeur reçoit la requête et trouve le bon contrôleur
    Route::get('/utilisateurs', [UserController::class, 'index'])
  3. Le Contrôleur traite la demande
    Il demande les données au Modèle
  4. Le Modèle récupère les données depuis la base
    User::all() → retourne tous les utilisateurs
  5. Le Contrôleur envoie les données à la Vue
    return view('users.index', ['users' => $users])
  6. La Vue génère le HTML final
    La page s'affiche dans le navigateur

Qu'est-ce que Composer ?

Le gestionnaire de dépendances PHP

📦 Définition
Composer est un outil qui permet d'installer et de gérer les bibliothèques (packages) PHP dont votre projet a besoin.

Pourquoi Composer ?

  • Télécharge automatiquement les packages nécessaires
  • Gère les versions et les dépendances
  • Charge automatiquement les classes (autoload)

Installation de Composer

Sur Windows : Téléchargez l'installateur sur getcomposer.org

Vérifier l'installation :

composer --version

Le fichier composer.json

Ce fichier liste toutes les dépendances de votre projet :

{ "require": { "php": "^8.2", "laravel/framework": "^11.0" } }

Créer un Nouveau Projet Laravel

Deux méthodes disponibles

Méthode 1 : Avec l'installateur Laravel (recommandé)

D'abord, installez l'installateur global :

composer global require laravel/installer

Puis créez votre projet :

laravel new mon-projet

Méthode 2 : Avec Composer directement

composer create-project laravel/laravel mon-projet
✅ Résultat

Laravel va télécharger tous les fichiers nécessaires et créer un nouveau dossier mon-projet avec l'application prête à l'emploi.

Lancer le serveur de développement

cd mon-projet
php artisan serve

Ouvrez votre navigateur à l'adresse : http://localhost:8000

Les Dossiers Importants

Ce que vous devez connaître

📁 app/

Contient le code principal de votre application :

  • Models/ : Classes qui représentent vos tables
  • Http/Controllers/ : Logique de traitement
  • Http/Middleware/ : Filtres des requêtes

📁 routes/

Définit les URLs de votre application :

  • web.php : Routes pour les pages web
  • api.php : Routes pour l'API

📁 resources/views/

Contient les fichiers Blade pour l'affichage :

  • Templates HTML avec syntaxe Blade
  • Extension : .blade.php

📁 database/

Gestion de la base de données :

  • migrations/ : Structure des tables
  • seeders/ : Données de test
  • factories/ : Génération de fausses données
⚠️ Important
Le dossier vendor/ contient les packages installés par Composer. Ne modifiez jamais son contenu !

Le Fichier .env

Configuration de l'environnement

Le fichier .env contient les paramètres spécifiques à votre environnement (local, production...).

# Nom de l'application APP_NAME=TaskFlow APP_ENV=local APP_DEBUG=true APP_URL=http://localhost # Configuration de la base de données DB_CONNECTION=mysql DB_HOST=127.0.0.1 DB_PORT=3306 DB_DATABASE=taskflow DB_USERNAME=root DB_PASSWORD= # Configuration du mail MAIL_MAILER=smtp MAIL_HOST=mailhog MAIL_PORT=1025
🔒 Sécurité
  • Le fichier .env contient des informations sensibles (mots de passe)
  • Ne jamais le partager ni le mettre sur Git
  • Il est automatiquement ignoré dans .gitignore

Artisan : L'Outil en Ligne de Commande

Votre assistant Laravel

Artisan est un outil puissant qui vous aide à effectuer des tâches courantes rapidement.

Commandes essentielles

Commande Description
php artisan serve Démarrer le serveur de développement
php artisan list Voir toutes les commandes disponibles
php artisan make:controller NomController Créer un nouveau contrôleur
php artisan make:model Nom Créer un nouveau modèle
php artisan migrate Exécuter les migrations
php artisan route:list Lister toutes les routes
💡 Astuce
Utilisez php artisan help commande pour avoir de l'aide sur une commande spécifique.

Nouveautés de Laravel 11

Ce qui change par rapport aux versions précédentes

🆕 Principales nouveautés

Structure simplifiée

  • Moins de fichiers de configuration
  • Dossier app/ plus léger
  • Un seul fichier bootstrap/app.php

PHP 8.2 minimum

  • Meilleures performances
  • Nouvelles fonctionnalités PHP

Nouveaux outils

  • Health checks intégrés
  • Rotation des clés simplifiée
  • Laravel Reverb (WebSockets)

SQLite par défaut

  • Plus besoin de MySQL pour démarrer
  • Idéal pour le développement local

📅 Support à long terme

Laravel 11 sera supporté jusqu'en février 2027 (corrections de bugs) et août 2027 (sécurité).
