Création de Formulaires

Structure de base avec Blade

Formulaire HTML avec protection CSRF

<form action="{{ route('projects.store') }}" method="POST"> @csrf {{-- Protection CSRF obligatoire --}} <div class="form-group"> <label for="name">Nom du projet</label> <input type="text" name="name" id="name" value="{{ old('name') }}" class="@error('name') is-invalid @enderror"> @error('name') <span class="error">{{ $message }}</span> @enderror </div> <button type="submit">Créer</button> </form>
Protection CSRF

La directive @csrf génère un token caché qui protège contre les attaques Cross-Site Request Forgery. Toujours l'inclure dans les formulaires POST, PUT, PATCH, DELETE.

Méthodes HTTP

PUT, PATCH, DELETE avec @method

Les navigateurs ne supportent que GET et POST. Laravel utilise un champ caché pour simuler les autres méthodes.

Formulaire de mise à jour (PUT/PATCH)

<form action="{{ route('projects.update', $project) }}" method="POST"> @csrf @method('PUT') {{-- ou @method('PATCH') --}} <input type="text" name="name" value="{{ old('name', $project->name) }}"> <button type="submit">Mettre à jour</button> </form>

Formulaire de suppression (DELETE)

<form action="{{ route('projects.destroy', $project) }}" method="POST" onsubmit="return confirm('Êtes-vous sûr ?')"> @csrf @method('DELETE') <button type="submit" class="btn-danger">Supprimer</button> </form>
old() avec valeur par défaut

old('name', $project->name) affiche l'ancienne valeur soumise en cas d'erreur, sinon la valeur actuelle du modèle.

Validation dans le Contrôleur

Méthode validate()

public function store(Request $request) { // Validation des données $validated = $request->validate([ 'name' => 'required|string|max:255', 'description' => 'nullable|string', 'status' => 'required|in:planning,active,completed', 'priority' => 'required|integer|between:1,5', 'due_date' => 'required|date|after:today', 'email' => 'required|email|unique:users,email', ]); // Si validation échoue : redirection automatique avec erreurs // Si validation réussit : créer le projet $project = Project::create($validated); return redirect()->route('projects.show', $project) ->with('success', 'Projet créé avec succès!'); }
Comportement automatique

Si la validation échoue, Laravel redirige automatiquement vers la page précédente avec les erreurs dans la session et les anciennes valeurs disponibles via old().

Règles de Validation

Les règles les plus utilisées

Règle Description Exemple
required Champ obligatoire 'name' => 'required'
string|max:n Chaîne, max n caractères 'name' => 'string|max:255'
email Format email valide 'email' => 'email'
unique:table,col Unique dans la table 'email' => 'unique:users'
confirmed Doit avoir un champ _confirmation 'password' => 'confirmed'
in:val1,val2 Valeur parmi la liste 'status' => 'in:draft,published'
exists:table,col Doit exister en base 'user_id' => 'exists:users,id'
date|after:date Date après une autre 'end' => 'after:start_date'

Syntaxe des Règles

Différentes façons d'écrire les règles

Syntaxe chaîne (simple)

$request->validate([ 'name' => 'required|string|max:255', 'email' => 'required|email|unique:users', ]);

Syntaxe tableau (recommandée)

use Illuminate\Validation\Rule; $request->validate([ 'name' => ['required', 'string', 'max:255'], 'email' => [ 'required', 'email', Rule::unique('users')->ignore($user->id), // Pour update ], 'status' => ['required', Rule::in(['draft', 'published'])], ]);
Règle unique lors de la mise à jour

Rule::unique('users')->ignore($user->id) permet de valider l'unicité en excluant l'enregistrement actuel.

Validation Conditionnelle

Règles dynamiques

Règles conditionnelles

use Illuminate\Validation\Rule; $request->validate([ // Requis seulement si un autre champ est présent 'city' => 'required_if:country,FR', // Requis sauf si... 'phone' => 'required_unless:contact_method,email', // Requis avec d'autres champs 'password' => 'required_with:password_confirmation', // Règle conditionnelle avec closure 'role' => [ 'required', Rule::when($request->user()->isAdmin(), 'in:admin,moderator'), ], ]);

Validation après d'autres règles

use Illuminate\Validation\Validator; $validator = Validator::make($request->all(), $rules); $validator->after(function ($validator) use ($request) { if ($this->somethingElseIsInvalid()) { $validator->errors()->add('field', 'Erreur personnalisée!'); } }); $validated = $validator->validate();

Validation de Tableaux

Valider des données imbriquées

Formulaire avec plusieurs éléments

<form method="POST"> @csrf @foreach($tasks as $index => $task) <input name="tasks[{{ $index }}][title]" value="{{ $task->title }}"> <input name="tasks[{{ $index }}][priority]" value="{{ $task->priority }}"> @endforeach </form>

Validation

$request->validate([ // Valider le tableau lui-même 'tasks' => 'required|array|min:1', // Valider chaque élément du tableau 'tasks.*.title' => 'required|string|max:255', 'tasks.*.priority' => 'required|integer|between:1,5', // Valider des tableaux imbriqués 'project.members' => 'array', 'project.members.*.user_id' => 'exists:users,id', ]);
Notation avec *

L'astérisque * permet de valider tous les éléments d'un tableau : tasks.*.title valide le titre de chaque tâche.

Règles Personnalisées

Créer vos propres règles de validation

Création d'une règle

php artisan make:rule Uppercase
<?php namespace App\Rules; use Closure; use Illuminate\Contracts\Validation\ValidationRule; class Uppercase implements ValidationRule { public function validate(string $attribute, mixed $value, Closure $fail): void { if (strtoupper($value) !== $value) { $fail('Le :attribute doit être en majuscules.'); } } }

Utilisation

use App\Rules\Uppercase; $request->validate([ 'code' => ['required', new Uppercase], ]);

Bonnes Pratiques

Recommandations pour les formulaires

À faire
  • Utiliser Form Requests pour la validation complexe
  • Toujours inclure @csrf
  • Utiliser old() pour conserver les valeurs
  • Valider côté serveur ET côté client
  • Personnaliser les messages d'erreur
  • Utiliser $request->validated()
À éviter
  • Ne jamais faire confiance aux données utilisateur
  • Ne pas oublier enctype pour les fichiers
  • Éviter les règles trop permissives
  • Ne pas utiliser $request->all() directement
  • Ne pas ignorer les erreurs de validation

Rappel sécurité

Toujours valider et nettoyer les données utilisateur. Utiliser $request->validated() ou $request->safe() pour ne récupérer que les données validées.
