1. Configuration de la Base de Données

Fichier .env

La configuration de la base de données se fait dans le fichier .env

Configuration MySQL

DB_CONNECTION=mysql DB_HOST=127.0.0.1 DB_PORT=3306 DB_DATABASE=taskflow DB_USERNAME=root DB_PASSWORD=

Configuration SQLite (Laravel 11 par défaut)

DB_CONNECTION=sqlite # Le fichier sera créé dans database/database.sqlite

SQLite

  • Pas de serveur à installer
  • Un seul fichier
  • Idéal pour le développement

MySQL

  • Plus performant
  • Multi-utilisateurs
  • Recommandé en production
Pour MySQL
Créez d'abord la base de données dans phpMyAdmin ou avec la commande : CREATE DATABASE taskflow;

2. Qu'est-ce qu'une Migration ?

Versionnement de la base de données

Définition
Une migration est un fichier PHP qui décrit les modifications à apporter à la structure de la base de données. C'est comme un "historique des changements" pour vos tables.

Pourquoi utiliser les migrations ?

❌ Sans migrations

  • Scripts SQL manuels
  • Difficile à partager
  • Pas de versionnement
  • Erreurs fréquentes

✅ Avec migrations

  • Code PHP versionné
  • Partageable via Git
  • Annulation possible
  • Équipe synchronisée

Emplacement des migrations

database/
    migrations/
        2024_01_01_000000_create_users_table.php
        2024_01_01_000001_create_cache_table.php
        2024_01_15_100000_create_projets_table.php
Format du nom
YYYY_MM_DD_HHMMSS_nom_de_la_migration.php
Le timestamp garantit l'ordre d'exécution.

Créer une Migration

Commande Artisan

Créer une migration pour une nouvelle table

php artisan make:migration create_projets_table

Structure du fichier généré

<?php use Illuminate\Database\Migrations\Migration; use Illuminate\Database\Schema\Blueprint; use Illuminate\Support\Facades\Schema; return new class extends Migration { /** * Exécuter la migration (créer la table) */ public function up(): void { Schema::create('projets', function (Blueprint $table) { $table->id(); $table->timestamps(); }); } /** * Annuler la migration (supprimer la table) */ public function down(): void { Schema::dropIfExists('projets'); } };
Méthodes up() et down()
  • up() : Ce qu'il faut faire (créer la table)
  • down() : Comment annuler (supprimer la table)

Modificateurs de Colonnes

Options supplémentaires

Schema::create('projets', function (Blueprint $table) { $table->id(); // Obligatoire (par défaut) $table->string('nom'); // Nullable (peut être vide) $table->text('description')->nullable(); // Valeur par défaut $table->string('statut')->default('en_attente'); // Valeur unique $table->string('email')->unique(); // Longueur personnalisée $table->string('code', 10); // Index pour recherche rapide $table->string('slug')->index(); // Combinaison $table->date('date_fin')->nullable()->index(); $table->timestamps(); });
Chaînage
Vous pouvez chaîner plusieurs modificateurs : ->nullable()->default('valeur')->index()

Exemple : Table Projets

Migration complète pour TaskFlow

public function up(): void { Schema::create('projets', function (Blueprint $table) { // Clé primaire $table->id(); // Informations du projet $table->string('nom'); $table->text('description')->nullable(); // Statut avec valeur par défaut $table->enum('statut', ['en_attente', 'en_cours', 'termine']) ->default('en_attente'); // Dates $table->date('date_debut')->nullable(); $table->date('date_fin')->nullable(); // Propriétaire du projet (clé étrangère) $table->foreignId('user_id') ->constrained() ->onDelete('cascade'); // Timestamps automatiques $table->timestamps(); }); }
Résultat
Cette migration crée une table projets avec toutes les colonnes nécessaires et une relation avec la table users.

Commandes de Migration

Exécuter et gérer les migrations

Commande Description
php artisan migrate Exécuter toutes les migrations en attente
php artisan migrate:status Voir l'état des migrations
php artisan migrate:rollback Annuler la dernière migration
php artisan migrate:reset Annuler toutes les migrations
php artisan migrate:fresh Supprimer toutes les tables et re-migrer
php artisan migrate:refresh Rollback + migrate
Attention
migrate:fresh et migrate:reset suppriment toutes les données ! À utiliser uniquement en développement.
Commande courante en développement
php artisan migrate:fresh --seed
Recréer toutes les tables et insérer les données de test.

3. Clés Étrangères

Lier les tables entre elles

Méthode moderne (Laravel 7+)

// Crée une colonne user_id + contrainte de clé étrangère $table->foreignId('user_id')->constrained(); // Avec action de suppression $table->foreignId('user_id') ->constrained() ->onDelete('cascade'); // Supprimer les projets si user supprimé // Référencer une autre table $table->foreignId('projet_id') ->constrained('projets') ->onDelete('cascade');

Actions de suppression

Action Comportement
cascade Supprimer les enregistrements liés
restrict Empêcher la suppression si des liens existent
set null Mettre la clé étrangère à NULL
Ordre des migrations
La table référencée doit exister avant ! Users doit être créé avant Projets.

4. Les Seeders

Remplir la base avec des données

Définition
Un seeder est une classe qui insère des données dans la base de données. Utile pour les données de test ou les données initiales.

Créer un seeder

php artisan make:seeder ProjetSeeder

Exemple de seeder

// database/seeders/ProjetSeeder.php use App\Models\Projet; class ProjetSeeder extends Seeder { public function run(): void { Projet::create([ 'nom' => 'Site E-commerce', 'description' => 'Boutique en ligne', 'statut' => 'en_cours', 'user_id' => 1, ]); Projet::create([ 'nom' => 'Application Mobile', 'description' => 'App iOS et Android', 'statut' => 'en_attente', 'user_id' => 1, ]); } }

Exécuter les seeders

php artisan db:seed --class=ProjetSeeder

5. Les Factories

Générer des données aléatoires

Définition
Une factory génère des données fictives réalistes grâce à la bibliothèque Faker. Idéal pour créer des données de test en masse.

Créer une factory

php artisan make:factory ProjetFactory
// database/factories/ProjetFactory.php class ProjetFactory extends Factory { public function definition(): array { return [ 'nom' => fake()->sentence(3), 'description' => fake()->paragraph(), 'statut' => fake()->randomElement([ 'en_attente', 'en_cours', 'termine' ]), 'date_debut' => fake()->dateTimeBetween('-1 month', 'now'), 'user_id' => User::factory(), ]; } }

Utiliser la factory

// Créer 1 projet Projet::factory()->create(); // Créer 50 projets Projet::factory(50)->create(); // Avec un statut spécifique Projet::factory()->create(['statut' => 'termine']);

Bonnes Pratiques

Conseils pour bien gérer vos migrations

✅ À Faire

  • Une migration = une modification
  • Noms explicites pour les tables
  • Toujours définir la méthode down()
  • Utiliser les clés étrangères
  • Créer des index sur les colonnes recherchées
  • Versionner les migrations avec Git

❌ À Éviter

  • Modifier une migration déjà exécutée en prod
  • Supprimer des migrations exécutées
  • Oublier les contraintes de clés étrangères
  • Mettre des données dans les migrations
  • Ignorer l'ordre des migrations
Astuce de développement

Pendant le développement, utilisez php artisan migrate:fresh --seed pour repartir de zéro. En production, créez une nouvelle migration pour chaque modification.
