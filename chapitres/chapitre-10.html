Tests

Introduction aux Tests

Pourquoi tester ?

Avantages des tests
  • Détection précoce des bugs
  • Confiance lors des refactorings
  • Documentation du code
  • Meilleure qualité globale
  • Facilite le travail en équipe

Types de tests Laravel

Type Objectif
Unit Tester une classe/méthode isolée
Feature Tester des fonctionnalités complètes
Browser Tester l'interface (Dusk)

Structure des tests

tests/
    Feature/        // Tests fonctionnels (HTTP, BDD)
    Unit/           // Tests unitaires (classes isolées)
    TestCase.php    // Classe de base
    Pest.php        // Config Pest (optionnel)

Performance

Éviter le Problème N+1

Eager Loading des relations

Mauvais : N+1 requêtes
$projects = Project::all(); foreach ($projects as $project) { // 1 requête par projet! echo $project->user->name; } // = 1 + N requêtes
Bon : Eager Loading
$projects = Project::with('user') ->get(); foreach ($projects as $project) { // Déjà chargé! echo $project->user->name; } // = 2 requêtes seulement

Détecter les problèmes N+1

// app/Providers/AppServiceProvider.php use Illuminate\Database\Eloquent\Model; public function boot(): void { // En développement : déclenche une exception si lazy loading Model::preventLazyLoading(! app()->isProduction()); }

Sécurité

Sécurité de l'Application

Protections essentielles

Protections intégrées

  • CSRF : Token automatique via @csrf
  • XSS : Échappement avec {{ }}
  • SQL Injection : Eloquent/Query Builder
  • Mass Assignment : $fillable
  • HTTPS : Forcer en production

Bonnes pratiques

  • Valider toutes les entrées
  • Utiliser les Policies/Gates
  • Hasher les mots de passe
  • Ne jamais exposer .env
  • Mettre à jour régulièrement

Forcer HTTPS en production

// app/Providers/AppServiceProvider.php public function boot(): void { if (app()->isProduction()) { URL::forceScheme('https'); } }

Déploiement

Déploiement sur un Serveur

Checklist de déploiement

  1. Cloner le projet sur le serveur
    git clone https://github.com/user/taskflow.git
  2. Installer les dépendances
    composer install --optimize-autoloader --no-dev
  3. Configurer l'environnement
    cp .env.example .env && php artisan key:generate
  4. Exécuter les migrations et optimiser
    php artisan migrate --force && php artisan optimize
  5. Configurer les permissions
    chmod -R 775 storage bootstrap/cache

Maintenance

Mode Maintenance

Pendant les mises à jour

Activer le mode maintenance

# Activer avec message personnalisé php artisan down --with-secret="token-secret" --render="errors::503" # Accéder malgré le mode maintenance (avec le secret) # https://taskflow.com/token-secret # Désactiver php artisan up

Tâches planifiées (Cron)

// routes/console.php (Laravel 11) use Illuminate\Support\Facades\Schedule; Schedule::command('backup:run')->daily(); Schedule::command('queue:prune-failed')->daily(); Schedule::command('telescope:prune')->daily(); // Sur le serveur (crontab -e) // * * * * * cd /path-to-project && php artisan schedule:run >> /dev/null 2>&1

Checklist

Checklist Avant Production

Ne rien oublier

Configuration
  • APP_ENV=production
  • APP_DEBUG=false
  • APP_KEY générée
  • BDD production configurée
  • HTTPS forcé
  • Mail configuré
Optimisation
  • php artisan optimize
  • npm run build
  • composer --no-dev
  • Cache Redis/Memcached
  • Queue worker configuré
  • Cron pour scheduler
Sécurité
  • Fichier .env NON versionné et sécurisé
  • Permissions correctes sur storage/ et bootstrap/cache/
  • Certificat SSL installé
  • Rate limiting activé
  • Monitoring d'erreurs configuré
