TP N°9 : Events, Notifications et Queues

Classe : MP1ILC
Enseignant : M. JARRAY Mondher
Durée : 1h30
Projet : TaskFlow

Objectifs du TP

  • Créer et dispatcher des événements (Events)
  • Implémenter des listeners pour réagir aux événements
  • Créer des notifications multi-canaux (email, base de données)
  • Configurer et utiliser les queues pour les tâches asynchrones
  • Afficher les notifications dans l'interface

Prérequis

  • TPs 1 à 8 complétés
  • Authentification fonctionnelle
  • Configuration email (Mailtrap recommandé pour les tests)

Partie 1 : Créer des Events (20 min)

Exercice 1.1 : Créer l'événement TaskAssigned

3 pts

Créez un événement qui sera déclenché quand une tâche est assignée à un utilisateur.

php artisan make:event TaskAssigned
// app/Events/TaskAssigned.php namespace App\Events; use App\Models\Task; use App\Models\User; use Illuminate\Foundation\Events\Dispatchable; use Illuminate\Queue\SerializesModels; class TaskAssigned { use Dispatchable, SerializesModels; public function __construct( public Task $task, public User $assignee ) {} }

Exercice 1.2 : Créer l'événement ProjectCompleted

2 pts
php artisan make:event ProjectCompleted
// app/Events/ProjectCompleted.php namespace App\Events; use App\Models\Project; use Illuminate\Foundation\Events\Dispatchable; use Illuminate\Queue\SerializesModels; class ProjectCompleted { use Dispatchable, SerializesModels; public function __construct( public Project $project ) {} }

Exercice 1.3 : Déclencher l'événement

2 pts

Ajoutez la colonne assigned_to à la table tasks et déclenchez l'événement.

php artisan make:migration add_assigned_to_to_tasks_table
// Migration Schema::table('tasks', function (Blueprint $table) { $table->foreignId('assigned_to') ->nullable() ->constrained('users') ->nullOnDelete(); });
// Dans TaskController ou TaskService use App\Events\TaskAssigned; public function assign(Task $task, User $user) { $task->update(['assigned_to' => $user->id]); // Déclencher l'événement TaskAssigned::dispatch($task, $user); return redirect()->back() ->with('success', 'Tâche assignée!'); }

Exercice 2.2 : Créer TaskAssignedNotification

4 pts
php artisan make:notification TaskAssignedNotification
// app/Notifications/TaskAssignedNotification.php namespace App\Notifications; use App\Models\Task; use Illuminate\Bus\Queueable; use Illuminate\Contracts\Queue\ShouldQueue; use Illuminate\Notifications\Messages\MailMessage; use Illuminate\Notifications\Notification; class TaskAssignedNotification extends Notification implements ShouldQueue { use Queueable; public function __construct( public Task $task ) {} // Canaux de notification public function via($notifiable): array { return ['mail', 'database']; } // Format Email public function toMail($notifiable): MailMessage { return (new MailMessage) ->subject('Nouvelle tâche assignée - TaskFlow') ->greeting("Bonjour {$notifiable->name},") ->line("Une nouvelle tâche vous a été assignée :") ->line("**{$this->task->title}**") ->line($this->task->description ?? 'Pas de description') ->action('Voir la tâche', route('taches.show', $this->task)) ->line('Merci d\'utiliser TaskFlow!'); } // Format Base de données public function toDatabase($notifiable): array { return [ 'type' => 'task_assigned', 'task_id' => $this->task->id, 'task_title' => $this->task->title, 'project_name' => $this->task->project->name, 'message' => "Nouvelle tâche assignée : {$this->task->title}", ]; } }

Exercice 2.3 : Créer le Listener

3 pts
php artisan make:listener SendTaskAssignedNotification --event=TaskAssigned
// app/Listeners/SendTaskAssignedNotification.php namespace App\Listeners; use App\Events\TaskAssigned; use App\Notifications\TaskAssignedNotification; use Illuminate\Contracts\Queue\ShouldQueue; class SendTaskAssignedNotification implements ShouldQueue { public function handle(TaskAssigned $event): void { // Envoyer la notification à l'utilisateur assigné $event->assignee->notify( new TaskAssignedNotification($event->task) ); } }
Découverte automatique

Laravel 11 découvre automatiquement les listeners grâce au type-hint dans la méthode handle().

Partie 3 : Configurer les Queues (20 min)

Exercice 3.1 : Configurer le driver de queue

3 pts

Pour le développement, utilisez le driver database.

php artisan queue:table
php artisan migrate

Modifiez votre fichier .env :

QUEUE_CONNECTION=database

Configuration email avec Mailtrap :

MAIL_MAILER=smtp MAIL_HOST=sandbox.smtp.mailtrap.io MAIL_PORT=2525 MAIL_USERNAME=votre_username MAIL_PASSWORD=votre_password MAIL_FROM_ADDRESS=noreply@taskflow.com MAIL_FROM_NAME="TaskFlow"

Exercice 3.2 : Lancer le worker de queue

2 pts

Lancez le worker dans un terminal séparé :

php artisan queue:work
Important

Le worker doit rester actif pour traiter les jobs. En développement, gardez ce terminal ouvert.

Pour voir les jobs en attente :

php artisan queue:monitor

Partie 4 : Afficher les Notifications (25 min)

Exercice 4.1 : Créer la vue des notifications

4 pts
