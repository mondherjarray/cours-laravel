TP N°4 : Modélisation Base de Données TaskFlow

Classe : MP1ILC
Enseignant : M. JARRAY Mondher
Durée : 1h30
Projet : TaskFlow

Objectifs du TP

  • Configurer la connexion à la base de données
  • Créer les migrations pour les tables de TaskFlow
  • Définir les relations entre les tables
  • Créer des seeders pour les données de test
  • Créer des factories pour générer des données

Prérequis

  • TP 1-3 terminés (projet TaskFlow avec routes et vues)
  • MySQL/MariaDB installé (Laragon, XAMPP, ou WAMP)
  • Comprendre les concepts de migrations (Chapitre 4)

Partie 1 : Configuration de la Base de Données (15 min)

Exercice 1.1 : Créer la base de données

2 pts

Option A : Via phpMyAdmin

  1. Ouvrez phpMyAdmin (http://localhost/phpmyadmin)
  2. Cliquez sur "Nouvelle base de données"
  3. Nom : taskflow
  4. Interclassement : utf8mb4_unicode_ci

Option B : Via le terminal MySQL

CREATE DATABASE taskflow CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

Exercice 1.2 : Configurer le fichier .env

2 pts

Ouvrez le fichier .env et modifiez les lignes suivantes :

DB_CONNECTION=mysql DB_HOST=127.0.0.1 DB_PORT=3306 DB_DATABASE=taskflow DB_USERNAME=root DB_PASSWORD=
Note

Si vous utilisez WAMP, le mot de passe par défaut est vide. Si vous utilisez MAMP, le mot de passe est souvent root.

Testez la connexion :

php artisan migrate:status

Si la connexion fonctionne, vous verrez un tableau vide (pas d'erreur).

Partie 2 : Création des Migrations (35 min)

Exercice 2.1 : Migration de la table Projets

5 pts

Créez la migration :

php artisan make:migration create_projets_table

Complétez le fichier créé :

public function up(): void { Schema::create('projets', function (Blueprint $table) { $table->id(); $table->string('nom'); $table->text('description')->nullable(); $table->enum('statut', ['en_attente', 'en_cours', 'termine']) ->default('en_attente'); $table->date('date_debut')->nullable(); $table->date('date_fin')->nullable(); $table->foreignId('user_id') ->constrained() ->onDelete('cascade'); $table->timestamps(); }); } public function down(): void { Schema::dropIfExists('projets'); }

Questions :

1. Pourquoi utilise-t-on ->nullable() pour la description ?

2. Que fait ->onDelete('cascade') ?

Exercice 2.2 : Migration de la table Tâches

5 pts

Créez la migration :

php artisan make:migration create_taches_table

Complétez la migration avec les colonnes suivantes :

  • id : clé primaire
  • titre : string, obligatoire
  • description : text, nullable
  • statut : enum (todo, in_progress, done), défaut 'todo'
  • priorite : enum (basse, moyenne, haute), défaut 'moyenne'
  • date_echeance : date, nullable
  • projet_id : clé étrangère vers projets (cascade)
  • user_id : clé étrangère vers users (nullable, set null)
  • timestamps : created_at et updated_at
