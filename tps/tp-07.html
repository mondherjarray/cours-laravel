Partie 1 : Installation de Laravel Breeze (25 min)

Exercice 1.1 : Installer Laravel Breeze

4 pts

Laravel Breeze fournit un système d'authentification complet et simple.

  1. Installez le package Breeze via Composer
    composer require laravel/breeze --dev
  2. Exécutez l'installation avec le stack Blade
    php artisan breeze:install blade
  3. Exécutez les migrations pour créer les tables nécessaires
    php artisan migrate
  4. Installez et compilez les assets front-end
    npm install && npm run dev
Note

Gardez npm run dev actif dans un terminal pendant le développement pour la compilation en temps réel.

Exercice 2.2 : Associer les projets aux utilisateurs

3 pts

Chaque projet doit appartenir à un utilisateur. Modifiez la migration et le modèle.

1. Créez une migration pour ajouter user_id :

php artisan make:migration add_user_id_to_projects_table
// Migration public function up(): void { Schema::table('projects', function (Blueprint $table) { $table->foreignId('user_id') ->after('id') ->constrained() ->onDelete('cascade'); }); }

2. Ajoutez la relation dans le modèle Project :

// app/Models/Project.php public function user(): BelongsTo { return $this->belongsTo(User::class); } // Ajouter user_id au $fillable protected $fillable = ['name', 'description', 'status', 'user_id'];

3. Ajoutez la relation inverse dans User :

// app/Models/User.php public function projects(): HasMany { return $this->hasMany(Project::class); }
php artisan migrate

// app/Models/User.php protected $fillable = [ 'name', 'email', 'password', 'role', ]; public function isAdmin(): bool { return $this->role === 'admin'; } public function isManager(): bool { return in_array($this->role, ['manager', 'admin']); }

Partie 4 : Policies (25 min)

Exercice 4.1 : Créer une Policy pour Project

4 pts

Les Policies permettent de définir qui peut faire quoi sur un modèle.

php artisan make:policy ProjectPolicy --model=Project

Complétez la Policy avec les règles suivantes :

// app/Policies/ProjectPolicy.php namespace App\Policies; use App\Models\Project; use App\Models\User; class ProjectPolicy { // Tous les utilisateurs connectés peuvent voir la liste public function viewAny(User $user): bool { return true; } // Peut voir si propriétaire ou admin public function view(User $user, Project $project): bool { return $user->id === $project->user_id || $user->isAdmin(); } // Tous peuvent créer public function create(User $user): bool { return true; } // Peut modifier si propriétaire ou admin public function update(User $user, Project $project): bool { return $user->id === $project->user_id || $user->isAdmin(); } // Seul le propriétaire ou admin peut supprimer public function delete(User $user, Project $project): bool { return $user->id === $project->user_id || $user->isAdmin(); } }

Exercice 4.2 : Utiliser la Policy dans le contrôleur

3 pts

Ajoutez les vérifications d'autorisation dans ProjectController.

// app/Http/Controllers/ProjectController.php public function show(Project $projet) { $this->authorize('view', $projet); return view('projets.show', compact('projet')); } public function edit(Project $projet) { $this->authorize('update', $projet); return view('projets.edit', compact('projet')); } public function update(UpdateProjectRequest $request, Project $projet) { $this->authorize('update', $projet); $projet->update($request->validated()); return redirect()->route('projets.show', $projet); } public function destroy(Project $projet) { $this->authorize('delete', $projet); $projet->delete(); return redirect()->route('projets.index'); }

Exercice 4.3 : Utiliser @can dans Blade

3 pts

Cachez les boutons d'action selon les permissions.

{{-- resources/views/projets/show.blade.php --}} <div class="actions"> @can('update', $projet) <a href="{{ route('projets.edit', $projet) }}" class="btn btn-primary"> Modifier </a> @endcan @can('delete', $projet) <form action="{{ route('projets.destroy', $projet) }}" method="POST"> @csrf @method('DELETE') <button type="submit" class="btn btn-danger" onclick="return confirm('Êtes-vous sûr ?')"> Supprimer </button> </form> @endcan </div> {{-- Afficher le rôle de l'utilisateur --}} @auth <p>Connecté en tant que : {{ auth()->user()->role }}</p> @endauth
