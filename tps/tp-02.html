Exercice 2.3 : Créer le contrôleur TacheController

Créez un deuxième contrôleur pour gérer les tâches :

php artisan make:controller TacheController --resource

Implémentez les méthodes suivantes :

  • index() : retourne une liste de 5 tâches (simulées)
  • show($id) : retourne les détails d'une tâche
  • store(Request $request) : affiche les données reçues en JSON
Structure d'une tâche
[ 'id' => 1, 'titre' => 'Créer la base de données', 'description' => 'Définir le schéma de la BDD', 'statut' => 'todo', // todo, in_progress, done 'priorite' => 'haute' // basse, moyenne, haute ]

Ajoutez la route resource dans web.php :

Route::resource('taches', TacheController::class);

Partie 3 : Organisation Avancée (20 min)

Exercice 3.1 : Groupes de routes

Organisez vos routes avec des préfixes :

// Routes publiques Route::get('/', function() { return 'Bienvenue sur TaskFlow'; })->name('home'); Route::get('/a-propos', function() { return 'À propos de TaskFlow'; })->name('about'); // Routes de l'application (préfixe /app) Route::prefix('app')->name('app.')->group(function() { Route::resource('projets', ProjetController::class); Route::resource('taches', TacheController::class); }); // Routes API (préfixe /api) Route::prefix('api')->name('api.')->group(function() { Route::get('/projets', [ProjetController::class, 'index']) ->name('projets.index'); Route::get('/taches', [TacheController::class, 'index']) ->name('taches.index'); });

Après avoir ajouté ce code, répondez :

1. Quelle est l'URL complète pour créer un nouveau projet ?

2. Quel est le nom complet de la route pour lister les tâches dans l'application ?
